💡

Tesis operativa: El paradigma maduró. Vibe coding puro sirve para prototipos desechables. Lo que entrega producto profesional es Spec-Driven Development: el Product Architect dirige al AI con specs ejecutables antes de generar código. Veinte minutos en una spec ahorran dos horas de iteración.

Los 7 principios no negociables

Aplican a las tres categorías de proyecto sin excepción.

1
Architect First
Antes de generar una línea de código, existe una spec. Si no hay spec, no hay build.
2
Validar antes de construir
Workflow rápido sin problema validado es solo velocidad para construir lo equivocado.
3
Frameworks sólidos
Stack opinado (Next.js, Supabase, Vercel) reduce el espacio de problema para el AI.
4
Educar al AI explícitamente
claude.md, .cursor/rules, AGENTS.md no son opcionales. El AI olvida convenciones cada sesión.
5
Scope estrecho por prompt
Un prompt = un cambio acotado. Prompts amplios producen regresiones invisibles.
6
Checkpoints obligatorios
Después de cada feature: probar, commit, documentar. Sin checkpoint se construye sobre algo roto.
7
Auditoría humana en seguridad
Código AI tiene 2.74× más vulnerabilidades (ICSE-SEIP '26). Pase de seguridad humano antes de producción es obligatorio.

Errores recurrentes a evitar

⚠️

Lista compilada de los hallazgos del paradigma 2025–2026. Cada uno cuesta caro la primera vez.

ErrorConsecuencia
Saltar la spec porque el cliente quiere ver algo rápido20 min de spec ahorran 2h de iteración de prompts
Iterar sobre código AI sin resetAcumula vulnerabilidades. Cada 3–5 features: pausa de limpieza
Prompts amplios ("Add filtering to the app")Regresiones en cascada. Scopear el componente específico
Confundir prototipo con MVPBolt/Lovable hacen prototipos. MVP real = Cursor + spec + QA
No documentar decisiones arquitectónicasEl AI las olvida cada sesión. claude.md es donde viven
Dejar que el AI elija el stackStack abierto produce inconsistencia caótica
Cerrar el proyecto sin handoverEl cliente queda atado por dependencia, no por valor
